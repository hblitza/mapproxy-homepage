Sie möchten MapProxy weiterentwickeln oder haben Sie einen Fehler gefunden? Hier finden Sie einige nützliche Informationen wie Sie weiter vorgehen können.

Fehler melden

Für Fehler haben wir einen so genannten Issue Tracker. Hier können Sie schnell und einfach neue Einträge erstellen und Ihre Probleme melden.

Quelltext

Offizielle MapProxy-Versionen werden auf der PyPI Projektseite von MapProxy veröffentlicht. Hier finden Sie auch ein Archiv das alle alten, offiziellen Versionen beinhaltet. Inoffizielle Versionen oder Beta-Releases finden Sie hier.

MapProxy nutzt die Source-Code-Verwaltung Git. Wenn Sie noch keine Erafhrung im Bereich Git haben, empfehlen wir Ihnen das kostenlose Buch Pro Git (englisch).

Das Haupt-Repository finden Sie unter http://github.com/mapproxy/mapproxy

Um eine Kopie von MapProxy-Repository zu bekommen müssen Sie den Quellcode "clonen":

git clone https://github.com/mapproxy/mapproxy

Wie kann ich etwas beitragen?

Patches können über die Mailingliste oder can post patches to the mailing list or create a new ticket. Eine weitere Möglichkeit ist das Erstellen eines Forks und senden Sie ihre Änderung in einem Pull-Request. Melden Sie sich einfach auf der Mailinligste wenn Sie Fragen haben.

Weitere Information

In der Dokumentation finden Sie ausführliche Informaitonen zum Thema Entwicklung. Hier werden viele Aspekte wie zum Beispiel der Quellcode, Dokumetnation und Tests ausführlich beschrieben.
